# 越谷市PRサイト 仕様書

## 1. プロジェクト概要

### 1.1 目的
越谷市の魅力を国内外に発信し、観光客の誘致、移住促進、地域ブランディングの向上を図るWebサイトを構築する。

### 1.2 ターゲットユーザー
- 観光を検討している国内外の旅行者
- 越谷市への移住を検討している個人・家族
- 越谷市でのビジネス展開を考えている企業
- 越谷市に興味を持つ学生や研究者
- 地域情報を求める市民

### 1.3 サイトコンセプト
「暮らしやすさと自然が調和する、活気あふれる越谷」をテーマに、越谷市の魅力を多角的に伝える。

## 3. 各ページの詳細仕様

### 3.1 トップページ

#### 構成要素
1. **ヘッダー**
   - ロゴ（越谷市のシンボル）
   - グローバルナビゲーション
   - 言語切替（日本語/英語）
   - 検索機能

2. **メインビジュアル**
   - フルスクリーンのスライダー（3-5枚）
   - 越谷市の代表的な風景（葛西用水、大相模調節池、花田苑など）
   - キャッチコピー表示

3. **ピックアップコンテンツ**
   - 注目の観光スポット（カード型レイアウト、3-4件）
   - 最新イベント情報（3件）
   - 季節の特集記事

4. **越谷の魅力（4つの柱）**
   - 自然・公園
   - 歴史・文化
   - グルメ・特産品
   - アクセスの良さ
   （各項目に画像とテキスト、詳細ページへのリンク）

5. **数字で見る越谷**
   - 人口、面積、アクセス時間などの統計情報
   - アニメーション表示

6. **SNS連携**
   - Instagram、Facebook、Twitterのフィード表示
   - ハッシュタグ #越谷の魅力

7. **フッター**
   - サイトマップ
   - お問い合わせ
   - プライバシーポリシー
   - アクセシビリティ方針
   - 著作権表示

## 4. 機能要件

### 4.1 基本機能

#### 多言語対応
- 日本語/英語の2言語対応
- 言語切替ボタン（ヘッダー配置）
- URL構造: /ja/, /en/

#### レスポンシブデザイン
- PC、タブレット、スマートフォンに最適化
- ブレイクポイント
  - PC: 1200px以上
  - タブレット: 768px - 1199px
  - スマートフォン: 767px以下

#### 検索機能
- サイト内全文検索
- 検索結果のページネーション
- 検索履歴（Cookie使用）

#### SNS連携
- SNSシェアボタン（各ページ）
- SNSフィード表示（トップページ）
- OGP設定

#### お気に入り機能
- スポットのブックマーク機能（LocalStorage使用）
- お気に入り一覧ページ

### 4.2 管理機能（CMS）

#### コンテンツ管理
- ニュース記事の登録・編集・削除
- イベント情報の管理
- スポット情報の管理
- 画像・動画のアップロード管理

#### ユーザー管理
- 管理者アカウント管理
- 権限設定（閲覧・編集・削除）

#### アクセス解析連携
- Google Analytics連携
- アクセスログの確認

### 4.3 パフォーマンス要件

- ページ読み込み時間: 3秒以内（PC）
- 画像の最適化（WebP対応）
- CDN利用
- キャッシュ戦略の実装

### 4.4 セキュリティ要件

- SSL/TLS対応（HTTPS）
- フォームのCSRF対策
- SQLインジェクション対策
- XSS対策
- 定期的なセキュリティアップデート

### 4.5 アクセシビリティ

- WCAG 2.1 レベルAA準拠
- 適切な見出し構造
- 代替テキストの設定
- キーボード操作対応
- 色のコントラスト比確保

## 5. デザイン要件

### 5.1 デザインコンセプト
- 親しみやすく、温かみのあるデザイン
- 越谷市のイメージカラーを活用
- 写真を多用し、視覚的な魅力を伝える
- シンプルで分かりやすいUI

### 5.2 カラーパレット
- メインカラー: 越谷市のイメージカラー（グリーン系）
- サブカラー: ブルー系（水辺のイメージ）
- アクセントカラー: オレンジ系（活気のイメージ）
- ベースカラー: ホワイト、ライトグレー

### 5.3 フォント
- 日本語: 游ゴシック、Noto Sans JP
- 英語: Roboto, Arial
- 見出し: ゴシック体
- 本文: ゴシック体（読みやすさ重視）

### 5.4 画像・動画
- 高解像度の写真使用
- 統一感のあるトーン・色調
- 動画コンテンツ（プロモーション動画）
- アイコン: オリジナルまたはMaterial Icons

## 6. 技術要件

### 6.1 フロントエンド
- HTML5
- CSS3（Sass/SCSS使用）
- JavaScript（ES6+）
- フレームワーク: React / Vue.js（検討）
- レスポンシブフレームワーク: Bootstrap / Tailwind CSS

### 6.2 バックエンド
- CMS: WordPress / Headless CMS（Strapi、Contentful等）
- サーバーサイド言語: PHP / Node.js
- データベース: MySQL / PostgreSQL

### 6.3 インフラ
- ホスティング: クラウドサーバー（AWS、GCP、Azure）
- CDN: CloudFlare / AWS CloudFront
- SSL証明書: Let's Encrypt

### 6.4 開発環境
- バージョン管理: Git（GitHub / GitLab）
- タスクランナー: npm scripts / Webpack
- テスト: Jest / Cypress（検討）

## 11. リスク管理

### 想定リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 開発遅延 | 高 | バッファ期間の確保、マイルストーン管理 |
| 予算超過 | 高 | 段階的リリース、優先順位付け |
| セキュリティ脆弱性 | 高 | 定期的なセキュリティ監査 |
| アクセス集中 | 中 | スケーラブルなインフラ設計 |
| コンテンツ不足 | 中 | 事前のコンテンツ計画策定 |
